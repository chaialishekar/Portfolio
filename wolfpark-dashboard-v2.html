Database Architecture
SQL Schema — 7 Normalized Tables + ERD
Revised schema: added vehicles table, lot_permit_types junction, and event_lot_map junction to fix SET anti-pattern and one-lot event limitation.

✦ Schema Improvements (3 Changes)
① lot_permit_types junction table
Replaces the SET column anti-pattern on parking_lots.permit_allowed. A SET column can't be properly indexed or joined — using a junction table makes permit-to-lot matching a standard indexed JOIN instead of FIND_IN_SET(), which can't use indexes.
② vehicles table added
Users need to register a vehicle plate to receive an allocation. Without this, you can't track which car is parked where — critical for enforcement, towing alerts, and the contractor permit use case.
③ event_lot_map junction table
Original schema had a single affected_lot_id FK on campus_events — a basketball game affects 4+ lots. A many-to-many junction handles this correctly.

Query 1 — Real-Time Availability by Permit Type
-- Uses lot_permit_types junction (replaces FIND_IN_SET) SELECT pl.lot_name, a.available_spots, a.status FROM parking_lots pl JOIN lot_permit_types lpt ON lpt.lot_id = pl.lot_id JOIN availability a ON a.lot_id = pl.lot_id JOIN users u ON u.permit_type = lpt.permit_type WHERE u.user_id = :uid AND a.available_spots > 0 AND a.status != 'FULL' ORDER BY a.available_spots DESC;
Query 2 — Event-Based Overflow Alert (Multi-Lot)
-- Uses event_lot_map junction for multi-lot events SELECT e.event_name, pl.lot_name, a.available_spots, e.impact_level FROM campus_events e JOIN event_lot_map elm ON elm.event_id = e.event_id JOIN parking_lots pl ON pl.lot_id = elm.lot_id JOIN availability a ON a.lot_id = pl.lot_id WHERE NOW() BETWEEN e.start_time AND e.end_time AND e.impact_level IN ('HIGH', 'MEDIUM') ORDER BY e.impact_level DESC;
Query 3 — Allocation Efficiency (25% Improvement KPI)
-- Tracks successful allocations today vs total requests SELECT COUNT(*) AS total_allocs, SUM(CASE WHEN status = 'ACTIVE' THEN 1 ELSE 0 END) AS successful, SUM(CASE WHEN status = 'EXPIRED' THEN 1 ELSE 0 END) AS no_shows, ROUND(SUM(CASE WHEN status='ACTIVE' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS efficiency_pct FROM allocations WHERE allocated_at >= CURDATE();
Query 4 — Vehicle Lookup for Enforcement
-- Emily's use case + contractor: verify plate → allocation SELECT u.name, u.permit_type, pl.lot_name, v.plate_number, al.expires_at, al.status FROM vehicles v JOIN users u ON u.user_id = v.user_id JOIN allocations al ON al.vehicle_id = v.vehicle_id JOIN parking_lots pl ON pl.lot_id = al.lot_id WHERE v.plate_number = :plate AND al.status = 'ACTIVE' AND al.expires_at > NOW();
